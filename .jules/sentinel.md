## 2024-05-16 - XSS Prevention in react-simple-code-editor
**Vulnerability:** Cross-Site Scripting (XSS) vulnerability was found in the `App.tsx` component when rendering code via `react-simple-code-editor`. The `highlight={code => code}` prop directly rendered user input into the DOM without sanitization, allowing arbitrary JavaScript execution.
**Learning:** `react-simple-code-editor` relies on the `highlight` prop to return safe HTML. Directly returning unescaped user input opens an XSS vector. While standard practice might suggest using DOMPurify, it strips out HTML tags entirely, breaking the editor's functionality by hiding code from the user.
**Prevention:** To prevent XSS while maintaining editor functionality, the input must be explicitly sanitized by replacing reserved HTML characters (`<`, `>`, `&`, `"`, `'`) with their corresponding HTML entities (`&lt;`, `&gt;`, `&amp;`, `&quot;`, `&#039;`) before passing the string to the editor. Use an `escapeHtml` function within the `highlight` prop.
