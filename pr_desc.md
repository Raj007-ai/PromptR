🎯 **What:** The testing gap addressed
This PR introduces test coverage for the `generateTTS` function in `services/geminiService.ts`. Previously, this function lacked tests, leaving the base64 decoding logic, modality configurations, and API interaction vulnerable to regressions.

📊 **Coverage:** What scenarios are now tested
1. **Happy Path:** Successfully returning an `ArrayBuffer` and validating the API request structure (model choice, `AUDIO` modality, and speech config).
2. **Missing Candidates:** Throws correctly when the API returns an empty candidates array.
3. **Missing Parts:** Throws correctly when the API response has candidates but missing parts.
4. **Missing Inline Data:** Throws correctly when the API returns text instead of audio data.
5. **API Errors:** Gracefully handles and propagates errors generated by the Google GenAI SDK.

✨ **Result:** The improvement in test coverage
We now have comprehensive coverage ensuring `generateTTS` safely decodes base64 strings and accurately communicates with the `gemini-2.5-flash-preview-tts` model, providing a robust safety net for future refactoring.
